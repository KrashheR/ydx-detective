# Промпт для Claude Design — «Особые архивы»

Ты — senior product UI/UX designer для игры «Где ложь? Симулятор детектива» на Yandex Games.

Нужно сверстать production-ready дизайн для новых экранов этапа 4: **«Особые архивы»** — витрина тематических паков расследований и косметики стола.

## Контекст проекта

Игра: data-driven insurance-investigation game. Игрок выбирает дело, изучает улики, ставит штампы на противоречиях и выносит вердикт.

Основная визуальная метафора: **папка расследования на столе детектива**.  
Аудитория 30–60 лет. Приоритеты: читаемость, спокойная иерархия, крупные зоны нажатия, физические бумажные материалы.

Запрещено:

- neon/casino/mobile-game эстетика;
- агрессивные скидки, таймеры давления, конфетти;
- fantasy UI как системная оболочка;
- мелкий текст и слабый контраст;
- скрывать бесплатный путь открытия контента.

Использовать только существующий визуальный язык:

- архивные папки;
- бумажные листы;
- картотека;
- чернильные штампы;
- корешки папок;
- пломбы, вкладки, описи, служебные ярлыки.

## Текущие новые компоненты в коде

Нужно спроектировать полный дизайн для:

1. `SpecialArchivesEntry`
   - компактная точка входа в левом сайдбаре desktop;
   - компактная точка входа в мобильном меню;
   - находится после daily case и до стандартных дел;
   - должна быть заметной, но не конкурировать с активным расследованием.
2. `ThematicPacksModal`
   - модальное окно/архивная витрина «Особые архивы»;
   - список паков;
   - детальная область выбранного пака;
   - список дел внутри пака;
   - список косметики/скинов;
   - коллекционный штамп;
   - действия покупки и rewarded-разблокировки.
3. Состояния кнопок и карточек:
   - первое дело бесплатно;
   - рекламная разблокировка доступна;
   - дневной рекламный лимит использован;
   - покупка доступна;
   - покупка в процессе;
   - покупка успешна;
   - покупка восстановлена;
   - весь пак завершён;
   - платформа/платежи недоступны;
   - прототипный режим: действия disabled.

## Пакеты для прототипа

Использовать три оригинальных пака:

1. «Архив необычных происшествий»
   - закрытая академия прикладных практик;
   - лаборатории, редкие коллекции, необычные страховые случаи;
   - стиль: необычная, но всё ещё архивная папка, сургучная лента, латунная лампа.
2. «Полярный протокол»
   - удалённая научная станция;
   - снег, радиосвязь, генератор, полевые журналы;
   - стиль: холодный корешок, полевой журнал, служебные отметки станции.
3. «Особняк на утёсе»
   - наследство, антиквариат, семейные заявления;
   - стиль: тёмная архивная папка, старая рамка портрета, описи имущества.

Важно: не копировать известные франшизы, символы, композиции или узнаваемые миры.

## Что нужно спроектировать

### 1. Desktop entry в левом сайдбаре

Сделай компактную папку/стопку папок:

- заголовок: «Особые архивы»;
- подпись: «Новые расследования и оформление стола»;
- статус: `Открыто 1 из 4`, `Первое дело бесплатно`, `Новое`;
- визуально должна читаться как физический объект;
- вся карточка кликабельна;
- высота компактная, чтобы не вытеснять daily case и список дел ниже fold.

### 2. Mobile entry

После daily case в мобильном меню:

- полноширинная компактная строка/папка;
- крупная зона нажатия;
- не использовать мелкие карточки-карусели;
- должна выглядеть как продолжение архива, не рекламный баннер.

### 3. Modal / каталог «Особые архивы»

Спроектировать desktop layout:

- header с названием и закрытием;
- 2–3 карточки паков;
- выбранный пак раскрывается детально;
- список дел внутри пакета;
- превью скинов;
- коллекционный штамп;
- блок действий.

Вариант visual metaphor: **выдвижной ящик картотеки / раскрытая архивная папка с разделителями**.

Карточка пакета должна показывать:

- тему;
- число дел;
- прогресс открытия;
- статус: `Новое`, `Первое дело бесплатно`, `Открыто 1 из 4`, `Куплено`;
- визуальную обложку папки;
- 1–2 выглядывающих документа/вкладки.

### 4. Detail panel

Для выбранного пака:

- крупная обложка;
- короткая завязка без спойлеров;
- список 4–5 дел;
- включённые скины;
- коллекционный штамп;
- прогресс;
- кнопки:
  - primary: «Купить архив целиком»;
  - secondary: «Открыть следующее дело за рекламу»;
  - helper text: «Дело останется открытым навсегда»;
  - tertiary: «Восстановить покупки» или «Вернуться».

Бесплатный путь должен быть видимым и честным, не хуже читаемым, чем покупка.

### 5. Mobile modal

Сделать адаптив:

- full-screen sheet или высокий bottom sheet;
- список паков сверху или первым экраном;
- detail раскрывается ниже;
- кнопки закрепить внизу только если они не перекрывают контент;
- все tap target минимум 44px.

### 6. RTL / Arabic

Показать RTL-версию ключевых экранов:

- entry;
- каталог;
- detail panel.

Проверить, что:

- порядок колонок и выравнивание корректны;
- закрытие и CTA не ломаются;
- длинные арабские строки не наезжают на соседние элементы.

## Состояния, которые нужно показать

Обязательно сделать отдельные состояния:

1. Новый игрок, первое дело бесплатно.
2. Бесплатное дело завершено, rewarded доступен.
3. Rewarded идёт.
4. Rewarded успешно открыл следующее дело.
5. Rewarded недоступен / ошибка.
6. Лимит рекламной разблокировки на серверный день использован.
7. Покупка в процессе.
8. Покупка успешна, весь пак открыт.
9. Покупка восстановлена.
10. Все дела пакета завершены.
11. Примерка скина активна до конца дня.
12. Platform unavailable / prototype mode: кнопки disabled, спокойное пояснение.

## Тексты UI

Использовать такие базовые RU-строки:

- Особые архивы
- Новые расследования и оформление стола
- Первое дело бесплатно
- Открыто {opened} из {total}
- Купить архив целиком
- Открыть следующее дело за рекламу
- Дело останется открытым навсегда
- Следующее открытие — завтра
- Реклама сейчас недоступна. Попробуйте позже.
- Покупка восстановлена
- Примерить оформление
- Доступно до конца дня
- В купленных архивах нет обязательной рекламы
- Состав дел
- Оформление
- Коллекционный штамп

Не использовать:

- «БЕСПЛАТНО!!!»
- «Успей купить»
- «Скидка только сейчас»
- countdown pressure
- красные тревожные badges.

## Design tokens

Ориентироваться на существующие Tailwind-токены:

- `bg`
- `surface`
- `surface-2`
- `border`
- `paper`
- `ink`
- `text-light`
- `text-muted`
- `text-dim`
- `accent`
- `success`
- `danger`
- `gold`
- `stamp`

Не хардкодить яркие цвета. Цветовые акценты паков должны быть приглушёнными и архивными.

Шрифты:

- `font-serif` для названий папок, документов, штампов;
- `font-sans` для системного UI;
- `font-mono` для инвентарных номеров, статусов, служебных отметок.

## Motion

Использовать спокойные Framer Motion transitions:

- hover/focus: папка слегка приподнимается;
- open modal: лёгкий подъём и fade;
- unlock success: снимается бумажная пломба / выдвигается вкладка дела;
- no confetti, no fireworks, no shaking pressure.

## Accessibility

Обязательно описать:

- focus trap в modal;
- закрытие по Esc;
- видимый focus;
- `aria-label` для закрытия;
- `role="dialog"` и `aria-modal`;
- tap targets минимум 44px;
- contrast suitable for age 30–60.

## Handoff

В финале дай handoff для React + Tailwind + Framer Motion:

- дерево компонентов;
- props для `SpecialArchivesEntry`;
- props для `ThematicPacksModal`;
- структура `ThematicPack`;
- state machine для покупки/rewarded/prototype states;
- где подключать в `App.tsx`, `LeftSidebar.tsx`, `MobileDeskMenu.tsx`;
- список новых i18n ключей;
- список аналитических событий:
  - `special_archives_open`
  - `special_pack_view`
  - `special_pack_purchase_start`
  - `special_pack_purchase_success`
  - `special_pack_purchase_error`
  - `special_pack_ad_offer`
  - `special_pack_ad_reward`
  - `special_pack_ad_error`
  - `special_skin_trial_start`

## Критерии приёмки

- «Особые архивы» видны на первом экране, но не конкурируют с daily case.
- Пакеты различимы визуально без чтения длинного текста.
- Бесплатный путь понятен и не спрятан.
- Покупка выглядит удобством, а не давлением.
- Витрина не выглядит рекламным баннером.
- Desktop, mobile и RTL читаемы.
- Все состояния из списка показаны.
- Можно напрямую передать handoff разработчику для реализации в текущем React/Tailwind коде.
