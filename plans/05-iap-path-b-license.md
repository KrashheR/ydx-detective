# План 05 — IAP Путь B: «Лицензия детектива» (одноразовый премиум, главный оффер)

**Состав (навсегда, non-consumable):**
1. Отключение интерстишлов.
2. Witness Canvass без просмотра рекламы.
3. Страховка серии: 1 пропущенный день в неделю не сбрасывает daily-стрик.
4. Эксклюзивный коллекционный штамп (поле `stats.collectibleStampIds` уже существует).

**Красная линия из ревью:** НЕ продавать буст награды/XP — сломает лидерборд (XP) и ощущение
честности. Ни один множитель `rewardEngine` лицензией не трогается.

---

## Этап 1 — Entitlement и персист (0.5–1 день)

1. `PlayerStats` + `ownedProductIds: string[]` (общее поле — пригодится путям A/C, не бинарный
   флаг). Производный хелпер `hasLicense(stats)` по константе
   `gameConfig.monetization.licenseProductId = 'detective_license'`.
2. **saveVersion bump + migrate** (`ownedProductIds: []`) — объединить с общим бампом 7→8 из
   планов 01/03, если пакет идёт вместе; иначе 8→9.
3. Покупка: экшен стора → `purchaseProduct('detective_license')` → при success добавить id,
   `persist(true)` (критичный момент — немедленная запись в облако). На буте —
   `restorePurchasedProductIds()` → merge в `ownedProductIds` (новое устройство/переустановка).
   Non-consumable: покупку не поглощать.
4. `payerFlag` в userParams (связка с планом 02, этап 4).

## Этап 2 — Эффекты (1–1.5 дня)

| Эффект | Где менять |
| --- | --- |
| Без интерстишлов | `App.tsx:313-332` — обе ветки `showFullscreenAd(transition, 'verdict')`: при `hasLicense` вызывать `transition()` напрямую. Не трогать `yandexSDK` (адаптер не должен знать про экономику) |
| Canvass без рекламы | `gameStore.buyHint` (`gameStore.ts:453-456`): при `hasLicense` вместо `showRewardedAd(...)` — сразу `reveal(0)`. Лимит 1 canvass/дело (`canvassUsed`) сохранить — иначе лицензия превращается в «раскрыть всё дело бесплатно» |
| Страховка стрика | `src/engine/streakEngine.ts` — чистая функция получает флаг `hasStreakInsurance`: разрыв в 1 день не обнуляет `streakCount`, но и не увеличивает (день не «зачтён», просто прощён); не чаще 1 раза за 7 серверных дней ⇒ нужно persisted-поле `lastStreakSaveServerDay: number \| null` (в тот же бамп сейва). Только серверное время (инвариант 3) |
| Эксклюзивный штамп | При покупке/restore добавить `'stamp_license'` в `collectibleStampIds`; ассет + каталог штампов |

Остальные rewarded-офферы (double reward, restore funds, daily unlock, архивные разблокировки)
лицензия **не** отключает — они добровольные и выгодны игроку; решение зафиксировать в доке.

## Этап 3 — Витрина и UX (1–2 дня)

1. Карточка оффера в стиле продукта: «удостоверение в кожаной корочке» (бумага/картон/тиснение,
   без «золотых VIP»-градиентов). Точки показа: магазин/стол + мягкий контекстный оффер после
   закрытия интерстишла («Устали от рекламы?» — не чаще 1 раза в N дел, порог в
   `gameConfig.monetization`).
2. Состояние «куплено»: бейдж лицензии у профиля, штамп в коллекции, оффер исчезает.
3. i18n ×5: название, 4 буллета состава, кнопка, состояние «активна».
4. Аналитика: существующие `product_view`/`purchase_start`/`purchase_success`/`purchase_error` +
   параметр `productId`; `license_offer_seen` с параметром `placement`.

## Этап 4 — Верификация

- Тесты: streakEngine (страховка: 1 пропуск прощён, 2 подряд — сброс, кулдаун 7 дней), store
  (canvass без рекламы при лицензии, лимит сохранён), миграция, restore-покупок мерджит id.
- Ручной прогон: dev-режим (payments == null) — оффер скрыт/деградирует, игра не ломается;
  Yandex-песочница — покупка, переустановка → restore.
- Доки: `docs/04` (страховка стрика), `docs/06` (payments, персист, миграция), `docs/03`
  (canvass). Цена — решение по рынку Яндекс Игр (ориентир: 2–3× цены среднего архива, это
  «навсегда»-оффер).

**Итого:** ~4–5 дней. **Зависимости:** нет жёстких; лучше после критичной аналитики (план 02),
чтобы сразу мерить конверсию оффера.
