## Рекомендуемый порядок (спринты)

1. **Спринт 1 — данные и дешёвые фиксы.** 02-analytics (критичный блок) + 1.5 + 1.1. Разблокирует
   решения по всему остальному на реальных цифрах; 1.5/1.1 — маленькие правки движка/стора.
2. **Спринт 2 — качество core-loop.** 1.6 (разбор + градуированный XP), 1.4 (выбор цели подсказки),
   1.7 (убрать декоративный гейт из UI). Остаток 02-analytics (воронка, техметрики).
3. **Спринт 3 — ретеншн-механики.** 2.3 «Досье» (дёшево, данные уже есть) → 2.5 «Кабинет»
   (материализует уровни, лечит 1.7) → 2.1 «Связать улики» (самая дорогая, только для экспертных дел).
3½. **Параллельно в любой момент — мобильный UX.** 06 (этапы 14–16 чеклиста) не зависит от
   механик/экономики и не трогает персист; ядро (sticky-вердикт + bottom-sheet улик) даёт
   наибольший прирост удобства на телефоне за минимум работы.
4. **Спринт 4 — монетизация.** 05 (Лицензия — главный оффер, инфраструктура платежей уже есть) →
   04 (сезонный архив: механики цепочек + контент от Codex).

## Сквозные правила для всех планов (из CLAUDE.md — нарушать нельзя)

- Смена формы персиста → бамп `GAME_CONFIG.saveVersion` (сейчас **7**) + ветка в `migrate()`
  (`src/services/persistence.ts`).
- Все константы тюнинга — только в `src/config/gameConfig.ts`, не в движках.
- Любая новая строка UI — `UIKey` в `src/i18n/ui.ts` + переводы на **все 5 языков** (ru, en, tr, ar, kk).
- Статика (JSON дел) и рантайм (PlayerStats) не смешиваются; Zod-схема (`caseSchema.ts`) в локстепе с типами.
- Дизайн: «папка на столе следователя», без неона/градиентов; только токены Tailwind, без hex.
- Ворота готовности каждого этапа: `npm run typecheck` && `npm test` + обновить соответствующий `docs/*.md`.
