# Synthetic playtest

Каркас запускает 10 независимых синтетических исследовательских сессий поверх реального UI.
Это источник гипотез, а не замена тестам с людьми: модель имитирует профиль и может не воспроизводить
реальные возрастные, моторные или когнитивные особенности.

## Архитектура

```text
personas.json + scenario.json
            |
            v
synthetic-playtest.mjs -> Playwright context per persona -> game URL
            |                    ^
            v                    |
        Responses API ---- screenshot + action trace
            |
            v
artifacts/playtests/<run>/
  manifest.json
  <persona>/events.jsonl
  <persona>/screenshots/*.png
  <persona>/report.json
  <persona>/report.md
  aggregate.json
  aggregate.md
```

Каждый контекст имеет отдельные cookies и LocalStorage. Агент получает только текущий скриншот,
сценарий, свою персону и историю собственных действий. Координаты клика проверяются против viewport.
Отчеты требуют наблюдаемого доказательства (`step` + описание); демографические объяснения без
поведенческого подтверждения помечаются как гипотезы.

## Запуск

1. Запустить игру: `npm run dev -- --host 127.0.0.1`.
2. Задать `OPENAI_API_KEY` и при необходимости `PLAYTEST_MODEL`.
3. Проверить конфигурацию: `npm run playtest:validate`.
4. Запустить: `npm run playtest:run -- --url=http://127.0.0.1:5173 --parallel=2`.

По умолчанию модель задается через `PLAYTEST_MODEL`; без переменной раннер завершится с понятной
ошибкой, чтобы выбор модели и стоимость не менялись незаметно. Для пробного запуска одной персоны:

```powershell
npm run playtest:run -- --url=http://127.0.0.1:5173 --agent=p05-target-methodical
```

Параллелизм по умолчанию равен 1. Значение 2–3 ускоряет прогон, но повышает пиковую нагрузку API.
Скриншоты могут содержать игровой контент, поэтому каталог `artifacts/playtests/` не следует
публиковать автоматически.

## Интерпретация

- `severity: blocker` означает, что сценарий нельзя продолжить.
- `high` — ломается ключевая механика или возникает высокая вероятность неверного решения.
- `medium` — заметное трение без блокировки.
- `low` — локальная проблема или улучшение.
- Повтор одной проблемы несколькими персонами повышает уверенность, но не превращает синтетическое
  наблюдение в статистику реальных пользователей.

Перед продуктовым решением критичные выводы следует проверить аналитикой или модерируемым тестом с
3–5 реальными участниками из затронутого сегмента.
