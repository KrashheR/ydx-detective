# Архитектурный чек-лист: как довести игру до top-tier уровня

Документ отделяет изменения движка и UX от сценарного контента. Codex должен сначала сопоставить пункты с реальной архитектурой проекта, затем внедрять их без создания параллельного приложения и без изменения стабильных case ID.

## P0. Новая модель точной дедукции

- [ ] Добавить `claimStatements[]` с атомарными тезисами и полями `id`, `text`, `stampable`.
- [ ] Сохранить `claim_main` как нестампуемый исходный текст для совместимости.
- [ ] Каждый штамп хранит `{ caseId, statementId, evidenceId }`, а не только case/evidence.
- [ ] Каждая улика получает `statementLink.relation`: `supports`, `contradicts`, `contextualizes` или `reveals_season_clue`.
- [ ] Штамп лжи разрешён только для `contradicts` и только после просмотра/анализа улики.
- [ ] Подтверждающие и контекстные материалы нельзя случайно отметить противоречием.
- [ ] Экран заявления визуально делит текст на 2–4 кликабельных тезиса, не превращая его в анкету.
- [ ] После вердикта показывать точную пару «ложная фраза ↔ доказавшая улика».
- [ ] Награда за доказательства учитывает точность statementId, а не количество поставленных штампов.

## P0. Evidence graph вместо линейного чтения

- [ ] Поддержать `evidenceTier`: `core`, `supporting`, `bonus`, `arc`.
- [ ] `core` достаточно для честного вердикта; `bonus` повышает точность/награду, но не обязателен.
- [ ] `arc` не расходует обычный investigationBudget: он открывается после вердикта как сюжетная награда либо в отдельной фазе синтеза.
- [ ] Добавить опциональные зависимости `unlocksAfterEvidenceIds` и `revealsEvidenceIds`.
- [ ] Не раскрывать шесть прямых доказательств одновременно: hard-дело должно разворачиваться цепочкой.
- [ ] Показывать игроку понятный вопрос следующего шага: «Куда делся объект?», «Когда сделано фото?», «Кто открыл сейф?».
- [ ] Не подписывать карточки готовым выводом до открытия или завершения интерактива.
- [ ] `statusExplanation` показывать после анализа, штампа или вердикта согласно типу улики.
- [ ] Добавить soft-lock защиту: любое дело решаемо при самом неудачном допустимом выборе карточек.

## P0. Эталонный onboarding первых трёх дел

- [ ] Новый пользователь попадает в дело 1 сразу или максимум за один клик.
- [ ] До первого вердикта не показывать магазин, карьерную карту, дейлики, коллекции и длинные настройки.
- [ ] Дело 1 обучает: открыть улику → выполнить thermal scan → выбрать точную ложную фразу → отказать.
- [ ] Thermal-вывод дела 1 показывает время осмотра, прошедшие 440 минут и контрольное время остывания.
- [ ] Дело 2 обучает: подозрительная деталь не равна противоречию → одобрить без штампа.
- [ ] Дело 3 обучает: сравнить идентификатор → document scan → связать две улики → открыть Архив №17.
- [ ] После дела 3 одним коротким экраном открыть карьеру, daily case и доску Архива №17.
- [ ] Всё обучение встроено в реальные действия, пропускается опытным игроком и не содержит стен текста.
- [ ] Целевое время первой тройки: 8–12 минут.

## P0. Сложность и бюджет расследования

- [ ] У всех 50 дел должен быть числовой `investigationBudget`; `null` запрещён схемой.
- [ ] Easy: 2–3 карточки, одна ясная связь, без штрафа за onboarding.
- [ ] Medium: 3–5 карточек, одна ложная зацепка или необходимость связать два факта.
- [ ] Hard: минимум две правдоподобные версии, неполные доказательства и осмысленный выбор карточек.
- [ ] Количество карточек не считается сложностью само по себе.
- [ ] В актах 4–5 сохранить medium-передышки; не делать двадцать hard подряд.
- [ ] Подсказка уровня 1 напоминает принцип, 2 указывает тип данных, 3 подсвечивает область; ответ не выдаётся сразу.
- [ ] Первый бесплатный hint в каждом новом типе интерактива не снижает награду.

## P0. Финал кампании

- [ ] После правильного вердикта дела 50 открыть `finalSynthesis`.
- [ ] Перед финальным синтезом бесплатно открыть `evidenceUnlockIds` из arc-слоя; игрок не может потратить бюджет так, чтобы заблокировать финал.
- [ ] Реализовать универсальный JSON-driven компонент `EvidenceLinkBoard`, а не case-50-specific JSX.
- [ ] Игрок соединяет: форма 17-К → доли с выплат → маршрут B-17.
- [ ] Финальный вывод показывается только после двух правильных связей.
- [ ] После трёх неудачных попыток разрешить пропуск без блокировки завершения кампании.
- [ ] Сохранить отдельное событие аналитики и состояние завершения синтеза.
- [ ] Эпилог отображает последствия решений: Тернов, Илья, Аделия, Болат, Матеус и Вера.

## P0. Схемы, типы и миграция

- [ ] Нормализовать `gps` и `gps_track`: один канонический тип либо явный migration adapter.
- [ ] Обновить TypeScript union и Zod-схемы для новых полей без `any`.
- [ ] Добавить `superRefine`: уникальность statementId, существование statementLink, запрет contradiction на valid evidence.
- [ ] Проверять, что `investigationBudget <= evidence.length` и core-путь решаем в пределах бюджета.
- [ ] Стабильные case ID, evidence ID и ключи старых сохранений не менять.
- [ ] Старый штамп уровня case/evidence мигрировать к `claim_main` либо наиболее подходящему atom statement без потери награды.
- [ ] Завершённое старым пользователем дело не заставляет проходить новую улику ради уже полученной награды.
- [ ] Версионировать контент и прогресс интерактивов: `contentVersion`, `schemaVersion`, `revisionVersion`.

## P0. Ассеты и загрузка

- [ ] Не включать 110 изображений в initial bundle.
- [ ] На старте загрузить UI, портрет и ассеты дела 1; ассеты дела 2 предзагрузить во время первого расследования.
- [ ] После открытия дела N префетчить только N+1 и shared textures.
- [ ] Использовать WebP/AVIF с responsive `srcset`; хранить размеры, чтобы избежать layout shift.
- [ ] Критические надписи, даты, IMEI, подписи и температуры рисовать DOM/SVG.
- [ ] Удалять Canvas/WebGL-ресурсы при закрытии интерактива.
- [ ] Держать initial download желательно ниже 20 MB и время до gameplayStart ниже 10 секунд.
- [ ] Добавить placeholder/skeleton, но не показывать пустой чёрный экран.

## P0. Platform adapter для Yandex и CrazyGames

- [ ] Вынести рекламу, cloud save, locale, user profile, leaderboards и lifecycle в интерфейс `PlatformAdapter`.
- [ ] Реализации Yandex и CrazyGames не импортируются напрямую из компонентов игры.
- [ ] На CrazyGames удалить внешнюю рекламу, ссылки/брендинг других порталов и собственную fullscreen-кнопку.
- [ ] Вызывать `gameplayStart` в момент доступности первого реального действия, а не на splash screen.
- [ ] Для Full Launch подключить CrazyGames account/cloud save с LocalStorage fallback.
- [ ] Автоматически выбирать locale платформы и использовать English fallback.
- [ ] Проверить 907×510, 821×462, 800×450, 1280×720 и мобильный touch.

## P1. Retention-архитектура

- [ ] После дела 3 открыть доску пяти сюжетных фрагментов текущего акта.
- [ ] Daily dossier не блокирует кампанию и даёт отдельную награду за первое дело дня.
- [ ] Сохранять streak рабочих дней, но не обнулять его полностью за один пропуск.
- [ ] После возврата показывать одну строку recap и кнопку «Продолжить дело», без повторного меню.
- [ ] Сезонный teaser хранится в JSON и может вести к конкретному следующему делу.
- [ ] Добавить коллекцию раскрытых методов мошенничества: подмена объекта, ложная причина, завышенный масштаб, сговор, двойная продажа.
- [ ] Не использовать жёсткий таймер ожидания ради сюжета.
- [ ] Daily case, карьера и косметика не должны прерывать первые три дела.

## P1. Feedback и game feel

- [ ] Штамп имеет три стадии: наведение/прицеливание → удар → чернильный след с лёгкой вариативностью.
- [ ] Успешное открытие интерактивной детали даёт короткий визуальный фокус и безопасный haptic wrapper.
- [ ] Ошибка не блокирует управление длинной анимацией.
- [ ] Для `supports` использовать визуально иной, но не «зелёный = ответ» маркер.
- [ ] Звук можно отключить; ни одна механика не требует аудио.
- [ ] Respect `prefers-reduced-motion` и запретить тряску всего экрана.

## P1. Локализация

- [ ] Сначала утвердить RU master, затем профессионально адаптировать EN.
- [ ] После EN обновить TR, AR и KK без изменения фактов, времени и идентификаторов.
- [ ] Arabic RTL проверяется на реальных карточках, ползунках, drag/rotate и штампах.
- [ ] Не переводить собственные коды 17-К, GC-17, B-17 и стабильные ID.
- [ ] В изображениях не должно быть критического локализуемого текста.

## P1. Аналитика и эксперименты

- [ ] События: case_start, evidence_open, evidence_complete, statement_stamp, verdict, case_complete, teaser_continue.
- [ ] Для интерактивов: timeSpent, attempts, hintLevel, resetCount, abandoned, success.
- [ ] Для onboarding: time_to_first_action, time_to_first_stamp, case_1/2/3 completion и menu_unlock.
- [ ] Для сложности: evidence path, unused budget, wrong statement, hint before churn.
- [ ] Воронка: load → first action → verdict 1 → case 2 → case 3 → return D1 → case 10.
- [ ] A/B-флаги находятся в конфигурации: cover, short/long prebrief, teaser CTA, first free hint.
- [ ] Не A/B-тестировать факты и правильный ответ дела.

## P1. Тесты и QA

- [ ] Unit: Zod, reward accuracy, budget, migrations, final synthesis, locale fallback.
- [ ] Component: mouse/touch, reset, hints, progress restore для пяти интерактивов.
- [ ] Integration: первые три дела от чистого профиля до открытия меню.
- [ ] Integration: valid-дело нельзя превратить в contradiction через UI.
- [ ] Integration: любой fraud имеет хотя бы один доступный точный штамп в пределах бюджета.
- [ ] Visual regression: document overlay, thermal zones, shadow origin, seal snap, surface masks.
- [ ] E2E: завершение каждого акта, возврат после перезагрузки, финальный синтез.
- [ ] Performance: low-end mobile, DPR 1, длинная сессия без роста памяти.

## Definition of Done

- [ ] Первые три дела проходят без меню и объясняют весь core loop действиями.
- [ ] Все 50 дел имеют атомарные statements, числовой budget и валидные links.
- [ ] Hard означает синтез, а не лишнее чтение.
- [ ] Финал требует последнего действия игрока.
- [ ] Initial load не содержит кампанию целиком.
- [ ] Yandex и CrazyGames работают через один platform contract.
- [ ] RU/EN готовы, остальные локали не ломают факты и RTL.
- [ ] Автотесты и ручной QA пройдены, а аналитика позволяет увидеть отвал на каждом шаге.
