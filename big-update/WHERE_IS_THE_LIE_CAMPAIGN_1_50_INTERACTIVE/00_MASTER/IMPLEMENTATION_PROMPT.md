# Прямой промпт для Codex: внедрение кампании 1–50

Ты работаешь в существующем проекте «Где ложь? Симулятор детектива» как senior TypeScript/React engineer, narrative designer и системный геймдизайнер. Не создавай демонстрационный проект и не ограничивайся отчётом: адаптируй этот архив к реальной архитектуре игры.

## Порядок

1. Найди реальные case-файлы, Zod-схемы, TypeScript-типы, evidence renderer, Zustand store, сохранения, сортировку, локализацию и аналитику.
2. Сохрани все стабильные case ID и прогресс пользователей. Используй `campaignOrder` только для нового порядка 1–50.
3. Импортируй каноническую русскую версию из `ALL_CASES_1_50_RU.json`, адаптируя поля к существующей модели.
3.1. До реализации прочитай `ARCHITECTURE_CHECKLIST_TOP_TIER.md`: пункты P0 являются частью Definition of Done, а не рекомендациями.
3.1.1. Сюжетные правила из `CAMPAIGN_BIBLE_RETENTION_ADDENDUM.md` имеют приоритет над ранней библией при расхождении.
3.2. Первые три дела должны запускаться как единый onboarding-flow; главное меню и мета открываются только после дела 3.
4. Добавь пять discriminated-union типов: `document_scan`, `thermal_scan`, `shadow_time_check`, `seal_match`, `surface_reveal`.
5. Создай пять переиспользуемых React-компонентов; ни один не должен содержать условия по caseId.
6. Добавь Zod-схемы, touch/mouse управление, reset, три уровня hint, сохранение analysisCompleted и RTL.
7. Результат анализа, anomaly labels и статус противоречия скрывай до выполнения successCondition.
8. Для relation=supports не разрешай штамп противоречия; такая улика подтверждает valid-версию.
8.1. Для relation=contextualizes не разрешай штамп: мотив и подозрение не являются доказанным противоречием.
8.2. Штамп хранит точный statementId; claim_main остаётся нестампуемым исходным текстом.
9. Сгенерируй только файлы с `generatedBy=image_model`. SVG, heat zones, тени и фрагменты создавай программно.
10. После утверждения каждого изображения откалибруй percent coordinates, тени и допуски в JSON.
10.1. Для surface_reveal используй Canvas mask, Pointer Events и `touch-action: none`; считай trace intersection на служебной маске 128×128 после stroke, а не на каждом кадре.
10.2. Не сохраняй полную маску каждого мазка в MVP: сохраняй discoveredTraceIds, revealPercentByTrace и analysisCompleted; завершённую улику открывай с success mask.
10.3. При обнаружении trace допустим короткий haptic feedback через доступный безопасный wrapper, но механика не должна зависеть от вибрации.
11. Обнови ru, затем en/tr/ar/kk без изменения фактов. Не запекай локализуемый текст в изображения.
12. Реализуй универсальный EvidenceLinkBoard и finalSynthesis дела 50 без условий по caseId.
13. Добавь аналитику и пройди все автоматические и ручные проверки.

## Компонентный контракт

Каждый компонент получает только evidence JSON и callbacks прогресса. Общий результат:

```ts
type InteractiveEvidenceProgress = {
  evidenceId: string;
  opened: boolean;
  analysisCompleted: boolean;
  discoveredAnomalyIds: string[];
  selectedContradiction: boolean;
};
```

## Запреты

- Не менять стабильные ID дел и существующие ключи сохранений.
- Не превращать интерактив в обязательную сложную мини-игру дольше 20 секунд.
- Не показывать вывод до взаимодействия.
- Не полагаться только на цвет и не требовать звук.
- Не генерировать полный тепловизионный растр: использовать процедурный overlay.
- Не генерировать половины пломбы отдельно: сохранять master и процедурный seed.
- Не доверять генератору критический текст документов.
- Не использовать чужие логотипы, персонажей, знаменитостей и узнаваемую атрибутику.

## Definition of Done

Все 50 дел проходят Zod; все statements и links валидны; budget числовой; порядок и старые сохранения не сломаны; первые три дела проходят до меню; пять компонентов работают mouse/touch; прогресс восстанавливается; result скрыт до success; finalSynthesis работает; RTL проверен; ассеты загружаются лениво; аналитика отправляется; build/test проходят; каждый вердикт выводится из доступных фактов.
