# Handoff: генерация изображений без потери прогресса

Этот архив можно передать в отдельный чат. Его задача — создавать только растровые файлы из `ASSET_TODO.json`, сохранять прогресс после каждой принятой картинки и возвращать новый полный ZIP.

## Стартовый промпт другому чату

```text
Ты продолжаешь производство ассетов игры «Где ложь? Симулятор детектива».
Сначала прочитай README.md, 00_MASTER/IMAGE_GENERATION_HANDOFF.md, ASSET_STYLE_GUIDE.md, IP_SAFETY.md и ASSET_TODO.json.
Обрабатывай только записи status=pending или status=needs_regeneration в порядке priority, затем case.
После каждой принятой картинки сохрани файл по archiveOutputPath и сразу обнови соответствующую запись ASSET_TODO.json.
Не генерируй generatedBy=frontend и не рисуй критические даты, номера, подписи или UI-текст внутри изображения.
Перед статусом completed визуально проверь композицию, отсутствие логотипов, водяных знаков, лишних пальцев/лиц и пригодность для механики.
Когда лимит генерации изображений закончится, немедленно остановись. Не отмечай несозданные файлы completed. Обнови сводку ASSET_TODO.json, упакуй весь архив вместе с generated-assets и выдай новый ZIP. Следующий чат должен продолжить только оставшиеся pending.
```

## Правила очереди

- Всего image-model задач в исходной очереди: 110.
- Приоритет `0`: ассеты первых трёх дел; их сделать и проверить раньше остальных.
- Приоритет `1`: интерактивные улики и сюжетные финалы блоков.
- Приоритет `2`: остальные замены и условные ассеты.
- Не перегенерировать `completed`, если файл существует и его SHA-256 совпадает с записью.
- `generate_if_needed` и `replace_if_needed` сначала сверить с существующим проектным ассетом; без доступа к нему оставить `blocked_existing_asset_check`, а не придумывать замену.
- Фактический файл хранить внутри этого архива по `archiveOutputPath`; при внедрении Codex переносит его в `productionPath`.
- Для каждой принятой картинки заполнить `status`, `attempts`, `outputFile`, `sha256`, `width`, `height`, `generatedAt`, `reviewNotes`.
- Разрешённые статусы: `pending`, `generating`, `needs_review`, `needs_regeneration`, `blocked_existing_asset_check`, `completed`.

## Проверка перед возвратом ZIP

- [ ] Ни один отсутствующий файл не помечен completed.
- [ ] Каждый completed-файл открывается, имеет ожидаемое разрешение и SHA-256.
- [ ] В изображениях нет реальных брендов, чужих персонажей, знаменитостей и водяных знаков.
- [ ] Документы не содержат критического AI-текста; он остаётся DOM/SVG-слоем.
- [ ] Для thermal нет запечённой тепловой палитры; для surface нет процедурного покрытия; для shadow геометрия пригодна для калибровки.
- [ ] В ASSET_TODO.json обновлены summary и lastUpdatedAt.
- [ ] Возвращается весь архив, а не только папка generated-assets.
