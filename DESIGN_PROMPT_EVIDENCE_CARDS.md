# Evidence Card Visual Redesign — Prompt for Claude Design

## Контекст проекта

**Игра:** «Где ложь? Симулятор детектива» — страховой детектив на Yandex Games.  
**Стек:** React + TypeScript + Tailwind CSS + Framer Motion. Сборка Vite.  
**Дизайн-метафора:** всё UI — это папка с делом на столе детектива. Физические бумажные документы, официальные бланки, чернильные штампы, архивные папки.  
**Целевая аудитория:** 30–60 лет. Высокая читаемость, минималистичный лейаут, крупные таргеты.  
**Запрещено:** неон, мультяшная графика, игровые/sci-fi градиенты, казино-эстетика.

## Где находится код

Весь рендеринг тела улики — в компоненте `EvidenceBody` внутри файла  
**`src/components/StampModal.tsx`** (строки 210–293).

Текущие типы улик (`EvidenceType` в `src/types/index.ts`):
```
'photo' | 'gps' | 'document' | 'witness_statement' | 'camera_recording' | 'usage_log'
```

Данные улики приходят как `Evidence`:
```ts
interface Evidence {
  id: string;
  type: EvidenceType;
  title: LocalizedString;          // { ru, en, tr, ar, kk }
  content: LocalizedContent;       // string | string[] per language
  isContradiction: boolean;
  contradictionExplanation: LocalizedString;
}
```

`content` для `gps` — это `string[]` (массив строк, каждая = "время — место").  
`content` для остальных — `string` или `string[]`.

## Задача

Переделать функцию `EvidenceBody` в `StampModal.tsx`, заменив placeholder-заглушки на богатые, визуально достоверные карточки для каждого типа улики.  
Все изменения — только внутри `EvidenceBody`. Интерфейс компонента не менять.  
Использовать только Tailwind-токены и `framer-motion` для анимаций.  
Не добавлять новые npm-пакеты.

### 1. `gps` — Официальный отчёт GPS-мониторинга

**Визуальная метафора:** распечатка из мониторинговой компании «GeoGuard Solutions», напечатанная матричным принтером на плотной белой бумаге.

**Структура карточки (сверху вниз):**

**Шапка документа:**
```
┌─────────────────────────────────────────────────────┐
│  ▣ GeoGuard Solutions                    КОНФ.      │
│  Отдел мониторинга транспортных средств             │
│  ──────────────────────────────────────────────     │
│  Запрос: СТР-2024-03471-А   Тип: Полный трек        │
└─────────────────────────────────────────────────────┘
```
- Логотип «GeoGuard Solutions» — моно, мелким шрифтом `font-mono`, серый, с иконкой-ромбом ▣ перед ним.
- Справа вверху красный штамп `КОНФ.` / `CONF.` (красная рамка, `font-mono`, размер ~8px).
- Серая линия-разделитель.
- Номер запроса и тип отчёта в одну строку — `font-mono text-[10px] text-ink/50`.

**Карта (центр — главный элемент):**
- Тёмно-синий (#0f172a) прямоугольник 160px высотой, `rounded-md`.
- На нём белыми тонкими линиями — схематичная сетка улиц (CSS `repeating-linear-gradient` по двум направлениям, цвет `rgba(255,255,255,0.07)`).
- Поверх сетки — жёлтая пунктирная линия маршрута (SVG `<polyline>` или `<path>`, `stroke="#F59E0B"`, `stroke-dasharray="4 3"`, `stroke-width="1.5"`).
- На маршруте 3–5 кружков-точек. Каждая точка — белый круг (`r=3`) с анимацией `pulse` у _последней_ точки.
- При hover/focus на точке — tooltip с временем и адресом из `content[i]`.
- Точки расставляются равномерно вдоль диагональной линии по SVG-координатам, без реальной геолокации (данные декоративные).
- Слева внизу внутри карты — белый `font-mono text-[8px]` текст `© GeoGuard 2024 • GPS ACC: ±3m`.

**Интерактивность карты:**
- `useState(activePoint)` — по умолчанию `null`.
- Клик на точку устанавливает `activePoint = i`.
- В таблице ниже строка с индексом `i` получает `bg-accent/10 border-l-2 border-accent` — «активная» подсветка.
- Плавный transition (`transition-colors duration-200`).

**Таблица записей (под картой):**
- Фон белый, `rounded-b-md border border-t-0 border-black/10`.
- Заголовок строки: `TIME`, `LOCATION`, `STATUS` — `font-mono text-[9px] text-ink/40 uppercase`.
- Каждая строка из `content` (array): разбить по `" — "` → `[time, location]`.
- `STATUS` — всегда `●  LOGGED`, зелёный (`text-success`), `font-mono text-[10px]`.
- Чередование строк: `bg-white` / `bg-black/[0.02]`.
- Активная строка (совпадает с `activePoint`): `bg-accent/10 border-l-2 border-accent`.

**Нижний колонтитул:**
- `font-mono text-[9px] text-ink/35`: `Документ сформирован автоматически. Печать: ____/____ Подпись: __________`.

### 3. `camera_recording` — Интерфейс видеоплеера / видеорегистратора

**Визуальная метафора:** экран системы видеонаблюдения 1990-х — матовый пластиковый корпус ЭЛТ-монитора.

**Структура карточки:**

**«Монитор» (внешний контейнер):**
- `bg-[#1a1a1a] rounded-[4px] p-2 border border-[#333]`
- Внутренняя рамка вставки: `border border-[#444] rounded-[2px]`

**«Экран» (viewport):**
- Тёмно-зелёный/серый фон `bg-[#0d0d0d]`, высота `h-[160px]`
- Лёгкий «scan-line» эффект: CSS background `repeating-linear-gradient(0deg, rgba(0,255,0,0.01) 0 1px, transparent 1px 2px)` поверх.
- В левом верхнем углу: `●  REC` — красная точка с CSS `animate-pulse` + `font-mono text-[9px] text-red-500`
- В правом верхнем углу: `CAM-04` — `font-mono text-[9px] text-green-400/60`
- Внизу слева: таймкод `23:41:07` — `font-mono text-[10px] text-green-400/80` — анимируется (можно статично)
- В центре экрана — текущий «кадр»: серый placeholder `bg-[#1c1c1c]` с иконкой 📹 и текстом `font-mono text-green-400/30 text-[10px]`

**Кадры-слайды:**
- `useState(frame: number)` от 0 до N-1, где N = количество строк в `content` (каждая строка = кадр).
- Placeholder фона кадра: диагональный серый паттерн, разный для каждого кадра (чередуй светлость).
- Текст кадра (из `content[frame]`) — `font-mono text-[11px] text-green-400` внизу «экрана» в полупрозрачной чёрной полосе.

**Панель управления (под экраном):**
```
[◀◀] [▶▶]   кадр 1 / 3
```
- `bg-[#222] rounded-b flex items-center justify-between px-3 py-1.5`
- Кнопки `[◀◀]` и `[▶▶]`: `font-mono text-[11px] text-green-400 hover:text-white disabled:opacity-30`
- По центру: `FRAME 01 / 03` — `font-mono text-[9px] text-green-400/50`
- При переключении: Framer Motion `AnimatePresence` с `initial={{ opacity: 0 }} animate={{ opacity: 1 }}` и быстрым fade (0.15s) — имитация смены кадра.

**Внизу снаружи монитора:**
- Маленький серый текст: `font-mono text-[8px] text-ink/30 text-center mt-1`  
  `HIKVISION DS-2CD2143G2-I • IP-КАМЕРА • ФОРМАТ H.264`

**Детали реализации:**

- `bg-[#e8eef5]` (очень светло-голубой, имитация подсветки), `rounded-md`
- Внутренняя тень: `box-shadow: inset 0 0 30px rgba(180,210,255,0.6)` — «свечение» лайтбокса
- Бортики по бокам: `border-4 border-[#c0c8d0]` с `rounded-md`

- `bg-[#1a1f26] rounded-sm` высотой `h-[180px]`, `overflow-hidden`

Лупа при hover:
- `useState(mousePos)` + `onMouseMove`
- Небольшой круглый `div` `w-16 h-16 rounded-full`, позиционированный абсолютно у курсора
- Внутри: `transform: scale(2.5)` того же SVG-слоя (через CSS `clip-path` + вложенный дублированный слой)
- `border-2 border-white/50 shadow-lg`
- Курсор: `cursor-crosshair`

Угловые данные снимка:
- Абсолютно позиционированный `div` в правом нижнем углу снимка
- `font-mono text-[8px] text-white/50 bg-black/40 px-1.5 py-0.5 rounded`
- Содержимое: берётся из последних строк `content` (если они содержат дату/код) или hardcode-заглушка

Подпись под снимком:
- `font-mono text-[10px] text-ink/60 text-center mt-2`

## Технические требования

1. **Только изменения в `EvidenceBody`** в `StampModal.tsx`. Не трогать интерфейс, пропсы, логику стампинга.

## Примечания по контенту

Данные в `content` — это **текст**, который детектив читает как детектив. Карточки не показывают «реальные» изображения — только стилизованные placeholder-ы с реальным текстовым содержимым. Всё визуальное богатство достигается через CSS/SVG/Tailwind, без реальных фотографий.

Текст противоречия (где именно ложь) — это **не** `content`. Он находится в `contradictionExplanation` и показывается **только после вердикта** (в `ResultSheet`). Карточки улики должны отображать только `content` — нейтральные данные, которые игрок интерпретирует сам.
