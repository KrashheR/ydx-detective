# Геймдизайн-ревью «Где ложь? Симулятор детектива»

> Отчёт от 2026-07-18. Статус: план/бэклог, не авторитетная документация (та живёт в `docs/`).

**Общая оценка: 8/10.** Сильная основа: чистая核-фантазия («я заметил ложь»), детерминированная
экономика, контент-как-данные, честная кривая сложности, заложенные IAP и аналитика. Основные
проблемы — несколько жёстких/запутанных механик и дыры в воронке метрик.

---

## 1. Спорные механики — убрать или переработать

### 1.1. Банкротство — убрать (или превратить в косметику)
Баланс ≤ 0 → жёсткий гейт за rewarded-видео. Для аудитории 30–60 это фрустрация: игрок «беднеет»
в основном из-за подсказок (за попытку разобраться) и штрафов. Наказывать блокировкой прогрессии
за трату валюты — антипаттерн. Альтернатива: баланс не уходит ниже 0, «восстановить средства за
рекламу» остаётся добровольным бустом.

### 1.2. Баланс в лидерборде — заменить на XP (баг + дизайн-ошибка)
В лидерборд публикуется `balance` — расходуемая валюта: купил подсказку → упал в рейтинге. Игрок
наказан за использование механики, на которой строится монетизация. Лидерборд называется `'xp'`
(`LEADERBOARD_NAME` в `yandexSDK.ts`), а отправляется `balance` — расхождение зафиксировано.
Публиковать нужно `xp`.

### 1.3. Двойная цена Inspector Note — упростить
Подсказка списывает 20% от `claimAmount` **и** показывает интерстишл. Платить дважды (валютой и
временем) — худший UX. Оставить одно: Inspector Note — только валюта (синк экономики),
Witness Canvass — только rewarded (уже так).

### 1.4. Подсказка раскрывает «следующую по порядку» карточку — дать выбор цели
Игрок не выбирает, что раскрыть, и платит за случайную с его точки зрения информацию — часто про
карточку, в которой уверен. Дать выбор цели подсказки: ценность и готовность платить вырастут.

### 1.5. Отрицательный итог при верном вердикте — сгладить
«Я был прав и потерял деньги» — плохое ощущение. Пол: `total ≥ 0` при верном вердикте; штраф
съедает бонусы, но не уводит в минус.

### 1.6. Неверный вердикт → жёсткий ноль — оставить, но подсластить
Идеологически понятно, но игрок, нашедший все противоречия и ошибившийся кнопкой, получает
столько же, сколько тыкнувший наугад. Минимум: в ResultSheet показать «вы нашли N/M
противоречий — вот что вы упустили» и градуировать XP за качество расследования (сейчас плоские
`wrongVerdictXp = 2`).

### 1.7. `requiredLevel` — фактически мёртвая механика
Реальный замок — строгая последовательность; уровень «всегда достижим». Двойной гейт, один из
которых декоративный. Либо убрать требование уровня из UI (оставить как внутренний тир
сложности), либо дать уровням реальную функцию (см. «Кабинет следователя»).

### 1.8. Тезисы заявления (§1, §2…) в модалке улики — непонятны игроку
Радиокнопки привязки улики к тезису заявления (`claimTheses` / `evidenceThesisLinks`) выглядят
как непонятные «пункты выбора» без пояснения, при этом неверная привязка молча превращает штамп
в ложный (штраф). Ни одно шипнутое дело сейчас `claimTheses` не задаёт — механика полумёртвая.
**Решение (2026-07-18): удалена целиком** — UI-радиокнопки, `claimTheses`/`thesisId` в типах и
Zod-схеме, `evidenceThesisLinks` в сессии, ветки в reward/mastery/store; `saveVersion` 6 → 7.
Поле `relation` в JSON дел оставлено как авторская метадата (движком не читается).

## 2. Новые механики

- **2.1. «Связать улики» (cross-reference).** Соединить две карточки, противоречащие друг другу
  (фото ↔ банковская выписка) — чистое детективное удовольствие, усложнение для экспертных дел.
- **2.2. Ставка на уверенность (wager).** Опциональный слайдер перед вердиктом: ставишь часть
  базовой награды, верный вердикт ×1.5–×2, неверный — теряешь ставку. Агентивность для опытных,
  ускоряет оборот валюты, новичкам не мешает.
- **2.3. Досье мошенников (коллекция).** Каждый разоблачённый — карточка в «Картотеке»: портрет,
  схема обмана, статистика. Дёшево (данные в JSON дел), долгосрочная цель, повод для реиграбельности
  (gold mastery = «полное досье»).
- **2.4. Еженедельное «Особое расследование».** Одно эксклюзивное сюжетное дело в неделю (ротация
  по серверной неделе, как daily). Крючок возврата после кампании, носитель сезонности.
- **2.5. Кабинет следователя.** Визуальная прокачка стола: штампы, лампа, табличка, грамоты за
  ачивки. Косметика в дизайн-языке «бумага и картон», витрина для косметических IAP, материализует
  уровни (лечит 1.7).

## 3. Три пути монетизации (IAP)

### Путь A — Контент (расширить существующие архивы)
1. «Сезонный архив» — проходной билет: покупка открывает поток из 10–12 дел, выходящих по 2–3 в
   неделю.
2. Ценовая лестница: маленький пак (3 дела) / большой архив / бандл «все архивы со скидкой».
Лучший фит для аудитории 30–60: платят за «ещё почитать», не за ускорение.

### Путь B — «Лицензия детектива» (одноразовый премиум, главный оффер)
Навсегда: отключение интерстишлов + Witness Canvass без рекламы + страховка серии (1 пропущенный
день в неделю не сбрасывает стрик) + эксклюзивный штамп. **Не** продавать буст награды — сломает
лидерборд и ощущение честности.

### Путь C — Валюта и наборы (третий приоритет)
Паки `balance` («Премия от управления»: S/M/L) + one-time стартовый бандл новичка (валюта +
первый архив со скидкой). Работает только после фиксов 1.2–1.4. Косметику из 2.5 продавать здесь
поштучно.

## 4. Аналитика — что добавить в Метрику

### Критичные
- **`caseId` в `verdict_submit`** — без него нет fail-rate по делам (главный инструмент
  балансировки контента).
- **`reject_blocked`** — показ тоста `rejectNeedsProof`: прямой сигнал непонимания core-механики.
- **`budget_exhausted`** — показ тоста исчерпанного бюджета.
- **Агрегаты рекламы в userParams**: `interstitialsSeenTotal`, `adsPerSession`,
  `verdictsSinceLastAd` — тюнить `AD_EVERY_N_VERDICTS` (3 — агрессивно) по фактическому
  churn-after-ad (поле `exitedAfterAd` уже есть — связать).
- **`locked_case_click`** — клики по закрытым делам/архивам: карта спроса на контент.

### Воронка и удержание
- Онбординг-воронка: `boot_complete` → `first_case_start` → `first_evidence_view` → `first_stamp`
  → `first_verdict` с таймингами.
- **`evidence_read_ms`** — время в модалке улики при закрытии: читают или кликают, какие улики
  перегружены.
- **`result_sheet_close`** — время на разборе + следующий шаг (следующее дело / стол / выход).
- **`daily_seen` vs `daily_claim`** — конверсия ежедневки.
- **`rating_prompt_shown`** — показы промпта (сейчас только действия).
- **`hint_offer_seen`** — видят кнопку подсказки vs покупают.

### Техметрики
`boot_duration_ms` (до `notifyGameReady`), `cloud_save_error` / `sdk_init_failed` (молчаливая
деградация сейчас невидима), `save_migrated` (с версий).

### userParams добавить
`campaignPosition` (точка отвала), `payerFlag`, `adsWatchedTotal`, `masteryGoldCount`.

## 5. Что нужно для 12/10

1. **Сюжетная мета-арка.** Повторяющийся антагонист, чьи схемы распутываются через несколько дел,
   финальное «большое дело». Только контент, движок не трогается.
2. **Явный туториал первым делом.** Механику «отклонение требует штампа» игрок сейчас открывает
   через тост-ошибку. Скриптованное дело-обучение с подсветкой.
3. **Звук.** «Чанк» штампа, шелест бумаги, скрип папки — половина ощущения «физического стола».
   Самый дешёвый прирост качества.
4. **LiveOps-ритм**: еженедельное дело (2.4) + сезонные архивы (путь A) + реальные A/B через
   `experimentGroup` (частота интерстишлов, цены подсказок).
5. **Социальный слой**: шаринг результата «вырезкой из газеты» («Я раскрыл дело за 4 улики из 6»).
6. **Закрыть раздел 1**: 12/10 — это не «больше фич», а когда ни одна механика не работает против
   игрока.

**Приоритет на ближайший спринт:** фикс 1.2 (лидерборд) + критичные метрики из раздела 4 —
дёшево и разблокирует решения по всему остальному на реальных данных.
