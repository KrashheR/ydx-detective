# Handoff: Экран «Дело» (Case File) — Claim Detective

## Overview
«Claim Detective» — браузерная игра для платформы **Yandex Games**. Игрок выступает страховым следователем, который проверяет подозрительные заявления о выплатах. Эмоция продукта: «я заметил ложь и разоблачил мошенника».

Этот пакет описывает экран **«Дело» (Case File)** и весь его цикл:
**Выбор дела → Изучение заявления → Штамповка противоречий в уликах → Вердикт → Разбор результата.**

Целевая аудитория: 30–60 лет. Приоритеты — высокая читаемость, минимализм, чёткая иерархия данных, крупные зоны нажатия, спокойный интеллектуальный темп. Запрещено: неон, мультяшная графика, игровые градиенты, sci-fi/fantasy, эстетика мобильных казино. Метафора — «папка-дело на столе следователя»: бумажные документы, официальные формы, чернильные штампы.

## About the Design Files
Файлы в этом пакете — **дизайн-референсы, сделанные в HTML** (прототипы, показывающие задуманный вид и поведение). Это **не продакшен-код для копирования**. Задача — **воспроизвести эти макеты в существующей среде вашего проекта** (React/Vue/Svelte и т.д.) по сложившимся в нём паттернам и библиотекам. Если среды ещё нет — выбрать подходящий фреймворк (для Yandex Games обычно React + CSS-in-JS или обычный CSS) и реализовать дизайн в нём.

Технологически прототип написан как «Design Component» (собственный мини-формат поверх React) — **не переносите этот формат**, берите только разметку, стили, тексты, цвета и логику.

## Gameplay Mechanics (логика)

### Система противоречий
Чтобы **отклонить** заявление, игрок обязан открыть улику и нажать «Отметить как противоречие». Если нажать «Отклонить» без единой отметки — показать тост:
> «Отказ должен быть обоснован. Отметьте хотя бы одно противоречие в документах.»

### Математика награды
- `base_verdict_weight = 0.50` — за верный вердикт (Одобрить/Отклонить).
- `evidence_proof_weight = 0.50` — пропорционально доле **реально найденных** противоречий из общего числа реальных (здесь 2: `receipt`, `gps`).
- Формула: `pct = (verdict==='reject' ? 50 : 0) + 50 * (realStamped / totalReal)`.
- Пример: верный отказ + оба противоречия = **100%**; отказ + 1 противоречие = **75%**.
- Отметка нейтральной улики (`photo`, `chat`) **не** даёт награды (ложный след — навык игрока).

### Правильное решение кейса «Угон BMW»
**Отклонить.** Воронцов инсценировал угон: его картой ••4417 в 02:47 заправлен его же автомобиль, а GPS-трекер зафиксировал выезд с парковки в 01:12 — тогда как по заявлению машина была угнана ночью, а он не выходил из дома. Одобрение = «ОШИБКА СЛЕДОВАТЕЛЯ», убыток компании 1 850 000 ₽.

### Daily Case
`rewardMultiplier ×5`, `complexityMultiplier ×2`, визуал: золотые рамки + текстурный штамп URGENT на обложке. В прототипе — тизер по таймеру.

### Локализация (i18n)
Data-driven, селектор языка в главном меню и левом сайдбаре (дропдаун с нативными названиями). Языки: `ru, en, tr, ar, kk`. UI прототипа на русском; вынести все строки в словари.

## Interactions & Behavior
- **Выбор дела:** клик по обычной папке → раскрытие дела (анимация `riseIn`, fade+translateY). Срочная/заблокированная → тост.
- **Открытие улики:** клик по карточке → модалка (`modalIn`, .28s, `cubic-bezier(.2,.9,.3,1)`); клик по бэкдропу или «✕» → закрытие; помечается «Просмотрено».
- **Штамповка:** тоггл-кнопка переключает отметку; штамп — анимация `stampIn` .45s `cubic-bezier(.2,1.3,.35,1)` (масштаб 2.6→1 с демпфером, rotate −13°).
- **Вердикт:** «Отклонить» без отметок → тост (auto-hide ~3.8s); иначе → результат; «Одобрить» → результат (здесь всегда ошибка).
- **Результат:** прогресс-бар анимируется к `pct` через ~110ms после открытия; «Следующее дело →» → сброс к выбору.
- **Табы (mobile):** переключение «Заявление»/«Улики».
- **Анимации (Framer Motion рекоменд.):** подъём карточек при ховере, плавный слайд документов, демпфер масштаба штампа. Длительности и easing — см. выше.
