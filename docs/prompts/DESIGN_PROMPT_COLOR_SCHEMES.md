# Бриф для Claude Design: 5 новых вариантов цветовых палитр

## Что за продукт

**«Где ложь? Симулятор детектива»** (*Where is the Lie?*) — страховой детектив-симулятор
для Yandex Games. Стек: React + TypeScript + Tailwind + Framer Motion, сборка Vite.

**Геймплей:** игрок изучает страховую претензию, открывает карточки-улики, штампует те,
что противоречат заявлению, и выносит вердикт «одобрить / отклонить». Награда зависит от
правильности вердикта и точности штамповки.

**Дизайн-метафора (жёсткое ограничение, не украшение):** экран — это *«Папка с делом,
разложенная на столе детектива»*. Физическая бумага, официальные бланки, чернильные
штампы, архивные манильские папки.

**Аудитория:** 30–60 лет. Приоритет — **высокая читаемость текста, минимализм, чёткая
иерархия данных, крупные зоны нажатия**, а НЕ зрелищность. Ядро эмоции: *«Я заметил ложь
и разоблачил мошенника»*.

**🚫 Красные линии — НИКОГДА не вводить:** неон, мультяшная графика, игровые/sci-fi
градиенты, фэнтези-элементы, эстетика мобильного казино. Любая палитра обязана остаться
«бумажно-архивной», читаемой при дневном свете, для взрослой аудитории.

## Как игра выглядит СЕЙЧАС — экраны

Десктоп — **три колонки «следственного стола»** (на мобильном схлопывается в одну колонку):

- **Левый сайдбар (~272px):** навигация по делам, прогресс кампании, выбор языка.
- **Центр (~480px):** папка с делом — обложка-манила с шапкой, бумажные документы внутри,
  сетка карточек-улик. По клику на улику открывается **модалка** с телом документа
  (рендереры: фото, GPS-трек, CCTV-запись, банковская выписка, детализация звонков,
  соцсеть, рентген, протокол свидетеля и т.д.) и кнопкой «Отметить как противоречие»
  (ставит красный чернильный штамп). Внизу — **панель вердикта** (Approve / Reject).
  После вердикта — **лист разбора награды** (ResultSheet).
- **Правый сайдбар (~272px):** бейдж уровня + XP-бар, серия дней, баланс, точность %,
  кнопка архива достижений, лидерборд.
- **Ежедневное дело:** премиальный вид — золотые рамки + штамп «URGENT» на обложке, ×5 награды.

Шрифты: **IBM Plex Serif** (документы/заголовки), **Inter** (UI-хром), **IBM Plex Mono**
(технические данные, коды, таблицы).

## Текущая палитра (Tailwind-токены — это и есть «расцветка»)

Цвета заданы как именованные токены в `tailwind.config.js`. Часть — фиксированный «хром
стола» (не зависит от темы папки), часть — CSS-переменные темы папки (`manila` / `dossier`).
Палитра = полный набор значений этих токенов.

**Фиксированный хром стола (тёплый «архив манилы»):**

| Токен | Сейчас | Назначение |
| --- | --- | --- |
| `bg` | `#ECE3D2` | фон стола (кремовая бумага) |
| `surface` | `#F6EFE1` | сайдбары / панели |
| `surface-2` | `#E2D7C2` | вложенные блоки внутри панелей |
| `border` | `#D2C4A8` | границы панелей |
| `ink` | `#1F2937` | текст на белой бумаге документа |
| `text-light` | `#3A3024` | основной текст на хроме |
| `text-muted` / `text-dim` | `#7A6C54` | вторичный / приглушённый текст |
| `success` | `#16A34A` | Approve |
| `danger` | `#DC2626` | Reject |
| `gold` | `#D9A441` | подсветка ежедневного дела |

**Тема папки (CSS-переменные, текущая тема `manila`):**

| Токен | manila | dossier | Назначение |
| --- | --- | --- | --- |
| `paper` | `#F5F1E8` | `#F3F4F6` | листы документов |
| `accent` | `#2F8F83` (бирюза) | — | основные действия / кнопки |
| `folder` / `folder-edge` | — | — | обложка папки и её тёмная шапка |
| `stamp` | `#B4231F` | `#DC2626` | чернильный штамп «противоречие» |

Скроллбар `#C9B89A`. Тосты тёмные (`#2B2018` + рамка `stamp`), чтобы читаться поверх стола.

## Задача

Предложи **5 новых цельных вариантов палитры**, которые были бы **«круче» и более
вовлекающими**, чем текущая, — но строго в рамках бумажно-архивной метафоры и красных
линий выше. «Вовлечённость» здесь = ощущение благородной, дорогой следственной атмосферы,
тактильность бумаги, удовольствие от чёткого контраста «улика / штамп / вердикт», а не
яркость ради яркости.

Для **каждого** из 5 вариантов дай:

1. **Название и настроение** (1 строка) — например «Полночный архив», «Сепия 1970-х»,
   «Холодный кабинет прокурора».
2. **Полная таблица токенов** с конкретными hex для ВСЕХ токенов выше
   (`bg`, `surface`, `surface-2`, `border`, `ink`, `text-light`, `text-muted`, `paper`,
   `accent`, `stamp`, `folder`, `folder-edge`, `gold`, `success`, `danger`). Это должно
   быть готово к вставке в `tailwind.config.js` / CSS-переменные.
3. **Обоснование** — почему этот набор повышает вовлечённость и удерживает метафору; чем
   он отличается от текущего тёплого «манилы».
4. **Контраст и читаемость:** подтверди, что текст на бумаге и на хроме проходит как
   минимум WCAG AA (для аудитории 30–60) — укажи ключевые пары (`ink` на `paper`,
   `text-light` на `surface`, кнопка `accent` с белым текстом).
5. **Как это смотрится на экранах** — короткое словесное превью трёхколоночного стола,
   обложки папки, карточки-улики и красного штампа в этой палитре (ASCII-мокап
   приветствуется).

Сделай 5 вариантов **разными по направлению** (например: один темнее/«ночной кабинет»,
один более холодный/официальный, один тёплый-сепия премиум, один контрастный/строгий,
один с необычным, но сдержанным акцентом вместо бирюзы) — чтобы было из чего выбрать.

В конце добавь **краткую сравнительную таблицу** всех 5 вариантов + текущей палитры
(настроение, акцентный цвет, светлая/тёмная) и **твою рекомендацию №1** с одним
предложением почему.
