# Визуальное ревью: живость и динамика интерфейса

> Ревью от Senior Product Designer. Основано на фактическом коде (Framer Motion / CSS
> transitions), а не на догадках — файлы и строки указаны везде, где это уместно.
> Метафора «папка на столе детектива» и красные линии из [docs/05-design.md](../05-design.md)
> (никакого неона/мультяшности/казино) — не нарушаются ни одной рекомендацией ниже.

## TL;DR

Игра **не статична равномерно** — есть один по-настоящему хороший, тактильный момент
(штамп улики) и несколько добротных микро-взаимодействий. Но вокруг них — большая
«мёртвая зона»: экономика (XP, баланс, точность), вердикт, награда и вся правая
панель обновляются **мгновенным подскоком чисел и CSS-заливкой полоски**, без единого
момента, который ощущался бы как «я это заработал». Для игры, где вся эмоция —
*«я заметил ложь и разоблачил мошенника»*, кульминация (клик по Reject/Approve →
экран награды) должна быть самым живым местом в игре. Сейчас это самое тихое место.

## Что уже хорошо (не трогать, использовать как эталон качества)

1. **Штамп противоречия** (`StampModal.tsx:168-188`) — `stampIn .45s
   cubic-bezier(.2,1.3,.35,1)`, лёгкий overshoot по scale, поворот -13°. Это ровно та
   тактильность, которой не хватает остальному интерфейсу — деревянный штамп,
   который «впечатывается» с отдачей.
2. **EvidenceCard hover/tap** (`EvidenceCard.tsx:37-43`) — `whileHover={{ y: -4 }}`,
   `whileTap={{ scale: 0.99 }}`, пружина `stiffness: 300, damping: 24`. Карточка
   улики физически «приподнимается», как настоящий лист. Хороший референс тайминга
   для всего остального.
3. **StampModal open/close** — fade фона + `y/scale` въезд листа улики с
   custom-bezier (`StampModal.tsx:99-120`).
4. **3D flip карточки** (photo/document face-down → face-up, `rotateY`,
   `StampModal.tsx:462-466`) и CCTV frame-cycling — приятные детали для конкретных
   типов улик.
5. **ResultSheet** уже использует motion для входа листа и печати штампа вердикта
   (`ResultSheet.tsx:156-225, 659-686`) — но см. ниже, чего в нём не хватает.

## Главные провалы (mertvaya zona)

### 1. Вся правая панель — статичные числа, без единого «момента признания»

`RightSidebar.tsx` — баланс, XP-полоска, точность, streak — используют только
`transition-[width] duration-500` на CSS-ширине бара (строки 78-83, 111-116). Само
**число** (`₽ {balance}`, `{accuracyPct}%`) меняется мгновенным React re-render:
не тикает, не подсвечивается, не даёт понять «только что +250₽ упало на баланс».

- **Баланс**: после ResultSheet баланс просто становится новым числом. Игрок не
  видит *дельту*. Нет ощущения «монеты падают в копилку».
- **XP-бар**: если прогресс переваливает за 100% и происходит level-up — это никак
  не выделено. Ранг просто меняется текстом (`levelTitle`). Повышение звания —
  один из самых мотивирующих моментов в progression-играх, а тут он неотличим от
  обычного тика прогресс-бара.
- **Точность %**: то же самое — просто новое число.

### 2. Вердикт-кнопки — самый важный клик в игре не даёт тактильного отклика

`VerdictPanel.tsx:39-67` — у кнопок Reject/Approve есть только
`transition-[filter]` + `hover:brightness-110`. Никакого `whileTap`, никакого
scale/press-эффекта. Сравните с тем, насколько лучше проработан hover карточки
улики (`y: -4`, пружина) — а тут кнопка, решающая исход дела, реагирует хуже,
чем карточка второстепенной улики.

### 3. DepartmentPlan / AchievementsModal / CaseSelect / Tooltip — 0 motion-импортов

Grep по `motion\.|whileHover|whileTap|animate=` не находит совпадений в
`DepartmentPlan.tsx`, `VerdictPanel.tsx`, `CaseSelect.tsx`, `Tooltip.tsx`,
`LeftSidebar.tsx`, `RightSidebar.tsx`. Это ~40% презентационных компонентов,
включая экран выбора дела (первое, что видит игрок при навигации) и апгрейды
отдела (единственный «shop»-момент экономики).

### 4. Нет ни одного звука во всём проекте

Grep по `Audio|sound` в `src/` — пусто. Ни звука штампа, ни «кассового» звука
при начислении награды, ни звука переворота страницы/открытия папки. Для игры,
где вся метафора — физическая бумага и чернильный штамп, отсутствие даже
минимального аудио-фидбека (глухой «тук» штампа, шелест бумаги) — самая
дешёвая по стоимости/эффекту недостача: 3-4 коротких sfx дадут больше ощущения
«живости», чем любая CSS-анимация.

### 5. ResultSheet: победа выглядит так же тихо, как поражение

Экран награды (`ResultSheet.tsx`) — самая эмоциональная точка игрового цикла
(«я разоблачил мошенника») — использует ту же plain fade/slide анимацию входа,
что и любой другой модальный экран. Числа награды (base/precision/streak
bonus/итог), судя по коду, не тикают по отдельности — просто рендерятся сразу
финальными.

## Рекомендации (приоритизировано по эффекту/стоимости)

### Уровень 1 — дёшево, высокий эффект

1. **Count-up для всех денежных/числовых значений.** Балансы, XP, точность,
   итоговая награда в ResultSheet — обернуть в лёгкий tick-хук (`useEffect` +
   `requestAnimationFrame`, ~400-600ms, ease-out), без новых зависимостей.
   Разница между «5 420 ₽» появившимся мгновенно и тем же числом, докрутившимся
   от 5 170 — это разница между «интерфейс обновился» и «я заработал деньги».
2. **`whileTap={{ scale: 0.96 }}` на VerdictPanel-кнопках** (и вообще на всех
   первичных CTA — DepartmentPlan upgrade-кнопках, achievements-кнопке). Копия
   паттерна, уже отработанного в `EvidenceCard.tsx:41-43`. Пара строк на
   компонент.
3. **Дельта-тост на балансе.** Короткая (~800ms) плавающая подпись `+250 ₽`,
   всплывающая рядом с балансом и уходящая вверх с fade-out, синхронно с
   началом count-up. Знакомый паттерн из mobile-economy UI, но без неона —
   просто моно-шрифт, цвет `success`, никакой частицы/конфетти.
4. **Разбивка ResultSheet на последовательные бисы (stagger).** Базовая
   награда → бонус за точность → streak-бонус → итог — сейчас, видимо, все
   строки появляются одним блоком. Даже 80-120ms `AnimatePresence` stagger
   между строками даст ощущение «чек распечатывается по одной строке», что
   прямо усиливает метафору официального документа.

### Уровень 2 — средняя стоимость, заметный эффект

5. **Level-up момент.** Когда `rank.level` увеличивается между рендерами —
   вспышка рамки XP-карточки (`ring`/`border` accent-цветом, 1 pulse), плюс
   отдельный маленький штамп-баннер «Повышение!» поверх RightSidebar на
   1.5-2 секунды. Это единственное действительно отсутствующее вознаграждение
   за долгосрочный прогресс.
6. **Полноценный sfx-слой.** 4-5 очень коротких (<300ms) звуков: штамп
   (глухой «тук», уже есть визуальная анимация под него), открытие папки дела,
   правильный вердикт (сдержанный, не аркадный «дзынь»), ошибка/reject без
   доказательств (короткий отказ), level-up. Держать в едином tasteful
   регистре — не игровые чиптюны, а «звуки офиса»: печать, бумага, скрепка.
   Обязательно с mute-переключателем и уважением системного звука Yandex
   Games (SDK context).
7. **CaseSelect / карточки дел — hover-приподнятие как у EvidenceCard.**
   Сейчас список дел — самый частый экран навигации — не имеет ни одной
   motion-анимации. Перенести туда же паттерн `y: -4` + spring.
8. **Progress-бар точности/XP — не просто width-transition, а лёгкий
   overshoot** (spring вместо ease), особенно в момент, когда полоска
   пересекает 100% и «переливается» в новый уровень — визуально свяжет рост
   полоски с самим level-up эффектом из п.5.

### Уровень 3 — более амбициозно, по желанию

9. **Page-turn переход между экранами** (CaseSelect → CaseFile → ResultSheet)
   вместо plain fade — лёгкий 3D-наклон/skew листа, продолжающий метафору
   бумажной папки (уже частично есть в 3D-флипе карточки улики
   `StampModal.tsx:462`, можно вынести как переиспользуемый паттерн перехода).
10. **«Живой» стол**: очень тонкая parallax/lift-тень при движении мыши на
    десктопе (не более 2-3px, только на крупных статичных панелях) — сугубо
    декоративный штрих, делает стол менее плоским. Низкий приоритет, только
    если остальное уже сделано — легко скатиться в излишество, которое
    конфликтует с «минимализм для аудитории 30-60».

## Что НЕ делать

Явно проговорить, раз тема — «эффекты и динамика»:

- Никаких частиц/конфетти/сияющих градиентов на level-up или победе — это
  прямое нарушение красных линий дизайн-языка (п. «Красные линии» в
  `docs/05-design.md`). Эмоция должна достигаться через **вес и тактильность**
  (пружины, count-up, звук штампа), а не через праздничные эффекты.
- Анимации не должны увеличивать время до интерактивности: count-up/stagger —
  максимум 400-700ms, whileTap — мгновенный отклик (spring, не delay).
- Не добавлять motion туда, где он будет мешать доступности/скорости чтения
  (аудитория 30-60, высокая читаемость — приоритет). Уважать
  `prefers-reduced-motion`, если ещё не уважается глобально — проверить перед
  раскаткой п. 9-10.
