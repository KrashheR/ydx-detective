# 📚 Документация проекта «Где ложь? Симулятор детектива»

Это центральный справочник по проекту. Каждый файл покрывает один срез: что это,
как устроено, как играется, как считается экономика, как выглядит, как
интегрируется с Yandex и как добавлять контент. Цель — быстро находить нужный
раздел, не перечитывая исходники.

> Для ИИ-агентов: `AGENTS.md` (Codex) и `CLAUDE.md` (Claude Code) в корне — это краткая
> карта + правила. Подробности
> живут здесь. Читай целевой файл из таблицы ниже вместо повторного вывода из кода.

## Связанные документы в корне

- `AGENTS.md` — правила для Codex + краткая карта проекта.
- `CLAUDE.md` — эквивалентные правила для Claude Code.
- `src/data/CASE_AUTHORING_GUIDE.json` — машиночитаемый гайд по генерации дел (схема,
  meta-поля, чек-листы, занятые значения). Первоисточник для авторинга — см.
  [07-authoring-content.md](07-authoring-content.md).
- `DEVELOPMENT_PLAN.json` — дорожная карта «Карьера следователя» (мета-прогрессия/экономика).
- `DESIGN_REQUIREMENTS.json` — исходный дизайн-бриф.
- `CODE_REQUREMENTS.json` — исходный архитектурный бриф.
- `docs/prompts/` — архив **одноразовых** дизайн-брифов (карточки улик, портреты, рейтинг-модалка).
  Не входят в карту доков и не подгружаются при обычной работе — см. [prompts/README.md](prompts/README.md).

## 🔄 Поддержание документации в актуальном состоянии (обязательно)

Документация — часть определения «готово». После **любого** изменения, меняющего поведение,
архитектуру, механики, экономику/конфиг, дизайн-токены, интеграцию с Yandex или процесс
авторинга контента, обнови соответствующий `docs/*.md` **в том же изменении**:

- Правь точечно — только затронутые строки, не переписывай файл целиком.
- Если сместилось необычное поведение или карта документации — поправь `AGENTS.md` и `CLAUDE.md`
  (раздел «Documentation map» и список non-obvious behaviors).
- Новый doc-файл → добавь его в таблицу карты выше, в `AGENTS.md` **и** в `CLAUDE.md`.
- Заметил, что doc уже разошёлся с кодом, проходя мимо — почини сразу.
- Соответствие: какой код → какой doc:
  движки/стор/типы → `02` (+ `03`/`04`); `gameConfig.ts` → `04`; UI/компоненты/Tailwind →
  `05`; `services/*` + persistence → `06`; новый контент/язык/тип улики → `07`; скрипты
  сборки/тестов → `08`.

## Соглашение об актуальности

Эта документация описывает состояние кода на момент написания. Если файл/функция/флаг,
упомянутые здесь, расходятся с исходником — **исходник первичен**; поправь документ.
Точки тюнинга экономики — `src/config/gameConfig.ts`, доменные типы — `src/types/index.ts`.
