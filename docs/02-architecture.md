# 02 · Архитектура и поток данных

> **🗺️ Ключевые файлы:** `src/store/gameStore.ts` (рантайм-авторитет), `src/App.tsx` (UI-контроллер), `src/engine/*.ts` (чистые движки), `src/services/persistence.ts`, `src/services/yandexSDK.ts`, `src/types/index.ts`.

## Правило №1, формирующее всё: разделение static / runtime

Две сущности строго разделены и **никогда не смешиваются**:

1. **Статические данные дел** — неизменяемый контент, авторизованный как JSON в
   `src/data/cases/`, валидируемый Zod при загрузке. Описан типами `Case` / `Evidence`
   в `src/types/index.ts`.
2. **Рантайм-состояние игрока** — изменяемая прогрессия/экономика, персистится в
   облако Yandex или LocalStorage. Живёт целиком в Zustand-сторе; описано
   `PlayerStats` / `ActiveSession` / `PersistedState`.

Никогда не клади `Case` внутрь `PlayerStats` и не клади прогресс игрока внутрь `Case`.
Данные дела **передаются в** экшены стора из React-слоя (после Zod-валидации) — стор
никогда не хранит контент дела. Слой персистенса сериализует только рантайм-срез
(`PersistedState`).

## Структура папок

```
src/
├── types/            # доменные интерфейсы (Case, Evidence, PlayerStats, …) + AssertAssignable
├── config/           # gameConfig.ts — тюнинг экономики/синка (чистые данные, без логики)
├── data/             # Zod-схема + JSON-дела + загрузчик + ачивки (контент)
│   ├── cases/        # стандартные дела (case-NNN.json)
│   │   └── daily/    # ежедневные дела (case-NNN-daily.json)
│   ├── caseSchema.ts # единственная граница доверия для контента (Zod)
│   ├── caseLoader.ts # реестр дел: импорт + валидация + выборки
│   ├── achievements.ts             # каталог ачивок (только метаданные + награда)
│   └── CASE_AUTHORING_GUIDE.json   # гайд по генерации дел
├── engine/           # чистая математика (награда/ранги/серии/ачивки/анлоки). Без SDK/стора
├── services/         # граница Yandex SDK + персистенс (cloud/local)
├── store/            # gameStore.ts — единственная рантайм-власть (Zustand)
├── i18n/             # ui.ts — строки интерфейса (5 языков, RTL для ar)
├── components/       # презентационные компоненты экрана «Дело»
├── utils/            # резолвер путей ассетов, форматирование подписей дел
└── App.tsx           # весь UI-контроллер: связывает стор с компонентами
public/covers, public/people   # обложки папок и портреты клиентов (SVG)
```

## Слои и зависимости

Стор (`src/store/gameStore.ts`) — единственная рантайм-власть. Он **компонует**
нижележащие слои, которые от него **никогда не зависят**:

- **`engine/rewardEngine.ts`** — чистый скоринг + математика доступности ежедневки.
  Без состояния/SDK/сайд-эффектов; детерминирован. Формула награды и кулдаун ежедневки.
  Модификаторы игрока (бонус % ранга/серии) **передаются аргументами** — движок чист.
- **`engine/rankEngine.ts`** — карьерная прогрессия: `evaluateXpGain` (дело → XP) и
  `evaluateRank` (кумулятивный XP → уровень + прогресс).
- **`engine/streakEngine.ts`** — серии: последовательные **серверные** дни → длина серии
  и ограниченный бонус.
- **`engine/achievementsEngine.ts`** — чистые предикаты разблокировки по id ачивки,
  вычисляемые против пост-кейс статов. Метаданные (i18n + бонусы) — в `data/achievements.ts`.
- **`engine/caseUnlockEngine.ts`** — гейтинг кампании: уровень-требование + строгая
  последовательность для стандартных дел; чистые селекторы статуса (available/locked/completed).
- **`services/persistence.ts`** — *где* лежит снапшот и *когда* он пишется. Не знает про
  дела/награды. Владеет миграцией версий сейва.
- **`services/yandexSDK.ts`** — **единственное** место, трогающее `window.YaGames`. Движок
  никогда не зовёт SDK напрямую; всё через этот адаптер, поэтому отсутствие/сбой SDK
  деградирует в офлайн-режим.

`App.tsx` — весь UI-контроллер: связывает стор с компонентами, управляет экранным
стейтом (выбранное дело, открытая модалка, result sheet) и **единственный** зовёт
экшены стора. Компоненты в `src/components/` — презентационные.

`BootScreen.tsx` — оболочка начальной загрузки. Она динамически импортирует `App`,
следит за готовностью SDK, гидратации, Zod-реестра и критических ресурсов и бесшовно
заменяет статический splash из `index.html` React-лоадером. В игровой lifecycle и
экшены стора оболочка не вмешивается. Сейчас оболочка временно отключена: `main.tsx`
монтирует `App` напрямую, а статический splash помечен `hidden`.

## Lifecycle-карта (продублирована в шапке gameStore)

```
init()              → initYandex() → loadSnapshot()      (гидратация: облако сначала)
UI hydrated         → notifyGameReady()                  (Yandex LoadingAPI.ready)
любая мутация       → persist()    → scheduleSync()      (облачная запись, дебаунс 10с)
closeCase()/вердикт → persist(true)→ flushSync()         (немедленная запись в облако)
открытие/закрытие   → onPauseChange() → setPaused()      (заморозка + мьют)
рекламы
restoreFunds()      → showRewardedAd() → onReward → сброс баланса
дейли-гейтинг       → getServerTimeMs()                  (НИКОГДА не время устройства)
```

## Реестр дел (`caseLoader.ts`)

Дела импортируются статически и собираются в `RAW_CASES`, затем валидируются
`safeParseCases`. Невалидные файлы **логируются и пропускаются**, в геймплей не
попадают. Выборки: `getAllCases`, `getCaseById`, `getStandardCases` (отсортированы по
критерию анлока), `getDailyCases` (детерминированно по id), `getDailyCase(dayIndex)`
(ротация пула по серверному дню).

## Связки, держащие схему и типы в синхроне

- `caseSchema.ts` (Zod) и `types/index.ts` (TS) синхронизированы compile-time проверками
  `satisfies`/`AssertAssignable`. Меняешь одно — меняй другое, иначе `typecheck` падает.
- `localizedShape` в схеме — `.strict()` и требует запись для **каждого** языка из
  `SUPPORTED_LANGUAGES`. Пропущенный перевод падает громко при загрузке.
- Уникальность id улик внутри дела гарантируется `superRefine`.
